# 入出力契約

現在の`handler.py`は、方向付き道路エッジ、経路コスト、出発・到着ノードを受ける内部fixture用の骨組みである。実道路やAWSには未接続であり、公開契約ではない。

本実装の公開契約は`POST /v1/routes`とする。入力は確定済みの出発・目的緯経度、探索モード、allow-list化した回避・優先条件、デモ基準時刻である。出力は最大3経路のGeoJSON、`segment_id`列、距離、時間、平均・最低指数、指数カバレッジ、信頼度、危険区間、グラフ・指数・設定版、`is_simulated`を含める。

詳細なJSON、エラー、候補生成規則は[雪道経路探索サービス設計書](../../../docs/architecture/route-planning-design.md)を正本とする。
