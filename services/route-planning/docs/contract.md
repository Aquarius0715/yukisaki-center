# 入出力契約

公開契約は`POST /v1/routes`である。入力は確定済みの出発・目的緯経度、探索モード、allow-list化した回避・優先条件、デモ基準時刻である。出力は最大3経路のGeoJSON、`segment_id`列、距離、時間、平均・最低指数、指数カバレッジ、信頼度、危険区間、グラフ・指数・設定版、`is_simulated`を含める。

Lambdaは16KiBを超える本文、長岡市外の地点、固定デモ時刻以外、未定義の回避・優先条件を拒否する。道路から100mを超える地点は400、グラフ・指数版が未ロードの場合は409、有効経路がない場合は404、一時的なDB障害は503を返す。

経路順位と数値はこのサービスが確定する。LLM説明が必要な場合は、このレスポンスを`POST /v1/ai/explain-routes`へ渡す。

詳細なJSON、エラー、候補生成規則は[雪道経路探索サービス設計書](../../../docs/architecture/route-planning-design.md)を正本とする。
